MyBatisPlus原理解析

发表于 2023-06-01|更新于2023-06-02|中间件
|字数总计:3168|阅读时长:14分钟|阅读量:

在使用MyBatis Plus时，我们的Mapper接口继承父接口：BaseMapper。BaseMapper中定义了一系列CRUD方法，子接口可以继承。但是项目中并没有编写 CRUD SQL语句，为什么可以对数据库进行CRUD操作呢？ MyBatis Plus是怎么进行SQL 语句拼接的？

分析

Mybatis Plus如何感知要操作哪张表，哪些列呢？

实体对象与表是可以进行映射的，在定义Mapper接口时，BaseMapper中明确指定了泛型为对应实体对象，那么可以借助反射获取实体类字节码对象。获取到字节码对象后，就可以通过反射获取实体类的类名和字段名。

SQL拼接需要考虑哪些核心点？

增：insert into 表(列1，列2 …) values(…)

删：delete from 表

改：update 表 set 列1 = xxx,列2 = xxx

查： select 列1，列2 from 表

  • 表名
  • 列名

二者对应MyBatis Plus中TableInfo类。

从Springboot自动装配类开始

从 mybatis-plus 源码中的 spring.factories 文件中我们可以了解到，其加载入口为 MybatisPlusAutoConfiguration。 点进去这个类可以发现里面有几个核心的 bean:

MapperScannerConfigurer

首先看MapperScannerConfigurer这个 bean，这是一个扫描 mapper 的配置类，实现了BeanDefinitionRegistryPostProcessor接口，是一个Bean工厂后处理器，内部使用 ClassPathMapperScanner 根据配置扫描 mapper 接口并将接口注册为 MapperFactoryBean。

创建 MapperScannerConfigurer bean的方式一般有两种:

  • 通过MyBatisPlus的自动装配类注册，默认扫描默认包下的带 @Mapper 注解的接口。
  • 通过解析 @MapperScan 注解的属性在 spring 中注册 MapperScannerConfigurer 的 beanDefinition，扫描@MapperScan注解设置的basepackage。

MapperScannerConfigurer#postProcessBeanDefinitionRegistry

创建 ClassPathMapperScanner 对象（继承 ClassPathBeanDefinitionScanner），设置相关属性（参考MapperScan属性），根据设置的属性注册扫描过滤器，开始执行扫描

ClassPathMapperScanner#doScan

  • 根据@MapperScan("com.example.mybatisplus.mapper")配置的basePackage扫描mapper，并注册 beanDifinition
  • 修改扫描到的 beanDefinition 类型为 MapperFactoryBean (或@MapperScan 指定的自定义类型)

扫描 Mapper 后会将其 beanDefinition.beanClass 修改为 MapperFactoryBean。 所以在后续 spring 将 mapper bean初始化时，会通过调用 MapperFactoryBean.getObject 获取其对象。

查看源码可以发现，最终是调用 sqlSession 的 getMapper 方法获取mapper代理对象。具体实现后面再看。

SqlSessionFactory

自动装配类里的sqlSessionFactory bean是 mybatis plus的用来获取 sqlSession (用来执行 sql 管理事务的对象)的工厂类。

创建 sqlSessionFactory 的逻辑在 MybatisSqlSessionFactoryBean#buildSqlSessionFactory 方法中，里面主要是

  • 设置配置文件里配置的属性：globalConfig，typeHandlers，interceptor…
  • 解析 mapper.xml 并保存到 configuration 中
  • 创建sqlSessionFactory

解析mapper.xml

XMLMapperBuilder 主要用于解析 mapper.xml 文件，入口为 XMLMapperBuilder#parse， XMLMapperBuilder 会解析 mapper.xml 文件中配置的 statements、resultMap、parameter 等信息， 并将其存放于 Configuration 的 对应 map 中。

解析 mapper.xml 后 Configuration 对象示例：当前只包含 mapper.xml 中的 sql 语句。

XMLMapperBuilder 解析完 xml 文件后，就会在 MybatisMapperRegistry 中添加对应的 Mapper 代理工厂 MybatisMapperProxyFactory。同时使用 MybatisMapperAnnotationBuilder 解析 mapper 接口中使用注解写的 sql 语句。

MybatisMapperProxyFactory 就是用来获取 Mapper 代理（MybatisMapperProxy）的工厂类。 在调用 Mapper 中方法的时候其实就是调用的 MybatisMapperProxy#invoke 方法。

MybatisMapperRegistry#addMappe

注入动态SQL

MybatisMapperAnnotationBuilder 在解析完注解 sql 后，会注入 mybatis-plus 的的 curd 动态 sql。平时调用 mybatis-plus 的 BaseMapper 的方法就是使用的这里注入的动态 sql。

abstractSqlInjector 注入的 sql 默认为 DefaultSqlInjector 中的方法（也就是 BaseMapper 中的方法）。 注入的方式为：根据不同方法的模板构建对应的 sql 脚本（同 xml），并填充对应的数据表和实体类的字段。这里的数据表字段是通过 TableInfoHelper 获得对应实体类的TableInfo类推断出来的，如根据 @TableField/@TableId 注解指定字段名，或根据驼峰下划线转换规则推断。

AbstractSqlInjector#inspectInject

ISqlInjector接口只有一个inspectInject方法来提供SQL注入的操作，在AbstractSqlInjector抽象类来提供具体的操作，最终对外的默认实现类是DefaultSqlInjector。

DefaultSqlInjector#getMethodList

以deleteById为例

SqlSessionTemplate

SqlSessionTemplate 实现了 SqlSession 接口，但是内部持有一个 sqlSessionProxy 代理对象，最后都是调用的代理对象的方法。 而代理对象中最终调用的 sqlSession 也是通过 sqlSessionFactory.openSession 来获取的。只不过在 openSession 前会从 spring 事务同步管理器中获取一遍，不存在才创建一个新的 sqlSession，并且再执行完成后关闭或释放（引用数量-1）。

这样就可以在需要使用 sqlSession 时直接使用 sqlSessionTemplate，而不是需要每次都通过 sqlSessinFactory 获取 sqlSession，也不需要考虑 sqlSesion 的关闭。同时保证了在同一个 spring 事务中使用同一个 sqlSession 对象。

获取 Mapper 对象

前面我们提到，将扫描到的Mapper接口将其 beanDefinition.beanClass 修改为 MapperFactoryBean。 所以在后续 spring 将 mapper bean初始化时，会通过调用 MapperFactoryBean#getObject 获取其对象注入到Spring容器中。

最终的调用链就是

MapperFactoryBean#getObject->SqlSessionTemplate#getMapper->configuration#getMapper->MybatisMapperRegistry#getMapper ->MybatisMapperProxyFactory#newInstance

前面解析 mapper 时提到了 MybatisMapperRegistry 提供了一个 getMapper 方法用来获取 mapper 代理对象。

在这个 getMapper 方法中之前注册的代理工厂 MybatisMapperProxyFactory 使用 通过 MybatisMapperProxy 生成了 mapper 的代理对象。

调用方法

总结

mybatisPlus 的整个加载过程概括如下：

  1. MapperScannerConfigurer 扫描 mapper 接口，并在 spring 中注册 beanDefinition，类型为 MapperFactoryBean
  2. SqlSessionFactory 解析 mapper.xml 和 mapper 接口 中的 sql 语句保存到 Configuration 中，同时加入 MybatisPlus 提供的动态 sql。 最后在 Configuration 注册 mapper 的 MybatisMapperProxyFactory。
  3. SqlSessionTemplate 使用 SqlSessionInterceptor 代理实现一个线程安全的 spring 管理的 SqlSession，并最终通过 MybatisMapperProxyFactory 获取 mapper 的代理对象 MybatisMapperProxy.
